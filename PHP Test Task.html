Hi, Congratulation on getting selected for the second round of the interview. In this round you are expected to deliver a small task. Please read the description of the task below.
Task description:
You are required to develop a simple web application. You can use an existing UI, there is no design for this. Before starting the task please create a simple GitHub Repository with the name bigo-php-task and email us the link at hr@bigosoft.co.uk
Due at 11:59
Commits after this time will not be considered. It’s very important that you commit and push your code regularly.
You will be judged based on the quality of your code, use of GitHub
This app should do the following:
As an applicant, I should be able to:
visit a website and submit the following information:
my name and email,
a URL linking to my LinkedIn profile (or other personal profile website)
an optional "cover letter" (as free text entered into a form field),
my resume as actual PDF file upload
view the status of my application (which should be one of "pending review", "ready to interview", or "archived").
if my application is still "pending review", I should be able edit my cover letter or upload an updated resume (but if the application is in a different status, I should no longer be able to edit/update anything).
As a hiring manager, I should be able to:
view a list of all the applicants who have applied who are still "pending review"
view the full set of available information for an applicant, including name, email, cover letter (if any), and resumé
click a button to update an application from the default status of "pending review" to one of "ready to interview" or "archived"
The list of applicants and the resume files should be persisted even if the user reloads a page or visits from a different browser. Applicants should not be able to see each other's applications.
